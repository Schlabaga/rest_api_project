
# Workshop Work Order Management API

 

Welcome to the documentation for the **Werkstatt Auftragsverwaltung (Workshop Work Order Management) API**. This REST API enables the comprehensive management of vehicle repair orders within a workshop environment.

## üìã Features

This API allows you to perform the following operations:

  * **Create** new repair orders.
  * **Retrieve** a list of orders with advanced filtering options.
  * **Read** details of specific orders.
  * **Update** order information and status.
  * **Delete** orders from the system.

## üöÄ Server

  * **Base URL (Dev):** `http://localhost:8080`

## üîê Authentication

The API uses **OAuth 2.0**.
To access protected resources, you must obtain an access token with the appropriate scopes.

| Flow | Scopes | Description |
| :--- | :--- | :--- |
| Implicit | `read:workorders` | Read-only access to work orders. |
| Implicit | `write:workorders` | Write access (create, update, delete). |

## üìö Endpoints

### 1\. Work Order Management

#### ‚û§ Get list of work orders

Returns a list of all work orders. Supports filtering via query parameters.

`GET /workorders`

**Query Parameters:**

| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `status` | string | No | Filter by status: `PENDING`, `IN_PROGRESS`, `COMPLETED`. |
| `licensePlate`| string | No | Filter by vehicle license plate. |
| `dueDate` | date | No | Filter by due date (`YYYY-MM-DD`). |

#### ‚û§ Create a new work order

Adds a new repair order to the system.

`POST /workorders`

**Request Body (JSON):**

```json
{
  "licensePlate": "SB-XY-123",
  "description": "Bremsscheiben wechseln",
  "status": "PENDING",
  "dueDate": "2025-10-15"
}
```

#### ‚û§ Get work order by ID

Retrieves the details of a specific work order.

`GET /workorders/{id}`

#### ‚û§ Update a work order

Updates the information of an existing work order.

`PUT /workorders/{id}`

#### ‚û§ Delete a work order

Permanently removes a work order from the system.

`DELETE /workorders/{id}`

## üì¶ Data Model

### WorkOrder

Represents a task/job for a vehicle.

| Property | Type | Description | Rules |
| :--- | :--- | :--- | :--- |
| `id` | integer (int64) | Unique ID (generated by server). | Read-only. |
| `licensePlate` | string | Vehicle license plate. | Max 20 chars. |
| `description` | string | Description of the work. | Max 255 chars. |
| `status` | string | Current status. | Enum: `PENDING`, `IN_PROGRESS`, `COMPLETED`. |
| `dueDate` | string (date) | Due date for completion. | Format: `YYYY-MM-DD`. |

**Full Response Example:**

```json
{
  "id": 101,
  "licensePlate": "SB-XY-123",
  "description": "Bremsscheiben wechseln",
  "status": "PENDING",
  "dueDate": "2025-10-15"
}
```
